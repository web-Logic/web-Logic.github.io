First Take on SVG Animations

  •   0 comments
  •   24/10/2016

CSS 3 introduced some cool animation options using just CSS. However I feel that they are very limited. There are only a limited number of events to which CSS animations respond. Moreover complex animations are just out of question. My primary reason to search for other animation tools is to create animated icons. Animated icons can be achieved through CSS but as I have mentioned earlier you can only do a bunch a different things (rotate, scale, change opacity, etc).

The options I considered

To be fair, I didn't search much about the alternatives available. I stumbled upon this Codrops article. There isn't much explanation available, moreover deciphering the code seemed like too much work. However I knew what I had to choose - SVG animated through snapsvg.js.

Apart from SVG animation, I also looked at GIF animations. However GIF animations have two big drawbacks:

  • They are created frame by frame, if I use Photoshop and Illustrator.
  • GIFs aren't the best quality available.

SnapSvg Library

SnapSvg is a comprehensive library for both creating and animating SVGs. I used SnapSVG only for animation purposes. For creating SVGs, I used Inkscape. Even though my goto program for vector graphics is Adobe Illustrator, I chose Inkscape because it generates cleaner SVG output.

My workflow while using Inkscape and Snapsvg can be summarised below:

  1. Create svg in Inkscape, give an id to each element of your SVG drawing.
  2. In your Javascript code, select the container element to which SVG tag will be appended.
  3. Provide url of the SVG file
  4. Assign a variable to the elements of the SVG drawing which will be animated.
  5. Animate!
